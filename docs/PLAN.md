# 골든타임: 물이 차오르기 전에 — 개발계획서

기상·기후 AI 해커톤 경진대회 개발계획서 양식(양식2)에 따라 작성한 대표 데모 개발 계획. 각 장은 공고 심사 5개 항목(주제 적합성 20 / 체험·참여형 설계 25 / 학습 효과 10 / 아이디어 독창성 15 / 실현 가능성 30)에 대응한다.

## 1. 프로젝트명

골든타임: 물이 차오르기 전에 — 집중호우·도시 침수 상황에서 대피 시간을 계산하고 안전한 경로를 판단하는 웹 기반 교육 게임

## 2. 개발 목적 및 필요성 (주제 적합성)

### 2.1 배경: 기존 재난 안전 교육의 한계

집중호우로 인한 도시 침수(반지하, 지하차도, 지하주차장)는 한국에서 매년 인명 피해가 발생하는 대표적 기상 재난이다. 그러나 기존 안전 교육은 영상 시청·수칙 암기 중심의 일방향 전달이어서, 실제 재난 상황에서 필요한 두 가지 능력 — "물이 차오르는 속도와 나의 대피 시간을 비교하는 시간 판단"과 "가까운 길과 안전한 길을 구분하는 경로 판단" — 을 훈련하지 못한다. 수칙을 아는 것과 골든타임 안에 판단하는 것은 다른 능력이다.

### 2.2 목적

기상 관측 데이터(강수강도)가 만들어내는 골든타임 안에서, 학습자가 스스로 대피 시간을 계산하고 경로를 선택·검증하는 체험을 통해 다음을 달성한다.

- 기상 데이터가 재난 위험(침수 속도)으로 이어지는 인과를 체험으로 이해
- "가장 가까운 길이 항상 가장 안전한 길이 아니다"라는 대피 원칙의 내면화
- 기상청 호우 행동요령(침수 도로·지하공간·하천변·맨홀 회피 등)의 실천적 학습

### 2.3 교육학적 기반

PhET(University of Colorado Boulder)의 연구 기반 설계 원칙(암묵적 스캐폴딩, 다중 표상, 안내된 탐구)과 Problem Based Learning을 결합한다. 해외 검증 사례인 일본 「쓰나미 대피 마스터」의 "이동 칸 수 = 대피 시간" 은유와 UNDRR 「Stop Disasters!」의 제한 조건 하 트레이드오프 판단 구조를 선별 반영한다.

## 3. 사용자 및 개발 형태

### 3.1 사용자

중학생(주 대상). 교육과정 연계: 2015 개정 중3 과학 「기권과 날씨」([9과18-01~04]) 및 「재해·재난과 안전」. 2022 개정 교육과정 [9과17-04] 탐구활동 "일기도, 레이더 및 위성 영상 등 실시간 데이터를 활용하여 날씨 해석하기"와도 직결된다.

### 3.2 개발 형태

게임형 웹 인터랙티브 시뮬레이션. 설치·로그인 없이 URL 접속만으로 심사위원 기기에서 즉시 시연 가능한 단일 웹앱. 한 차시(45분) 안에 완주 가능한 분량.

## 4. 활용 데이터 및 과학 개념

### 4.1 활용 데이터

기상청 지상관측(ASOS/AWS) 자료 — 기상자료개방포털(data.kma.go.kr) API.

- 강수강도(RN_INT, mm/h): 게임의 핵심 엔진. 강수강도 값이 도로 침수 속도, 즉 골든타임을 결정
- 강수량(RN)·일강수량(RN_DAY): 누적 침수 상황 설정
- 기온(TA)·습도(HM): 호우경보 상황판의 몰입용 보조 정보

데이터는 화면에 "시간당 빗물 42mm"처럼 학습자 눈높이 표현으로만 노출하고, 내부적으로는 API 응답과 동일한 필드 구조를 유지해 실데이터 연동을 보장한다.

### 4.2 과학 개념

집중호우와 강수강도, 강수강도와 지표 침수 속도의 관계, 저지대·지하공간의 침수 위험, 호우특보 기준.

## 5. 적용 기술 및 실현 구체성 (실현 가능성)

### 5.1 구현 방식과 기술 스택

- 프론트엔드: 단일 웹앱(HTML/CSS/JavaScript 기반, 프레임워크 확장 가능). 물 차오름 게이지·골든타임 카운터·갈림길 선택 UI를 표준 웹 기술로 구현
- 데이터 계층: 기상청 지상관측 API 응답 스키마와 동일한 구조의 데이터 객체를 두고, 강수강도를 골든타임으로 변환하는 규칙(30mm/h 미만 여유, 30~50mm/h 약 11분, 50mm/h 이상 약 8분)을 단일 함수로 분리. 가상 데이터에서 실제 API 호출로 교체가 한 지점에서 이루어지는 구조
- 배포: 정적 호스팅 기반 상시 접속 URL 제공

### 5.2 AI 도구 활용 방식

- 설계 단계: 생성형 AI로 학습 목표–평가 문항 정합성 검토, 시나리오(갈림길·피드백 문구) 초안 생성
- 개발 단계: AI 페어 코딩으로 게임 로직·UI 구현, 수치 균형(경로별 소요 시간과 골든타임의 정합) 사전 시뮬레이션 검증
- 검증 단계: 학생 예상 답안 생성으로 평가 문항의 오탐·누락 사전 점검

### 5.3 핵심 수치 구조 (사전 검증 완료)

게임 균형은 사전 계산으로 검증했다. 골든타임 12분(강수강도 42mm/h 기준) 하에서:

- 준비 1~2개 + 안전·빠른 경로: 여유 도착(잔여 약 4분)
- 안전·느린 경로: 빠듯한 도착(잔여 약 1분)
- 준비 과다(3개) + 느린 경로: 잔여 0분 아슬아슬 도착
- 위험 경로(침수 지하차도 등) 진입: 소요 시간과 무관하게 즉시 실패

이 구조로 "시간 계산", "골든타임 판단", "최단 경로가 곧 안전 경로는 아님", "위험 요소 식별"의 네 학습 목표가 모두 게임 규칙 안에서 작동한다.

### 5.4 개발 일정 (대회 일정 내 구현 계획)

1. 사전 개발(7.23~8.20): 핵심 게임 루프(상황–준비–갈림길–결과) 완성, 가상 데이터 기반 전체 플로우 동작 확인, 기상청 API 실데이터 연동
2. 중간 점검(8.10~8.12): 멘토 피드백 반영, 평가 문항 검증(예상 답안 기반 오탐·누락 점검), 학습자 눈높이 문구 다듬기
3. 본선(8.21~8.22): 최종 프로토타입 완성 및 URL 시연

### 5.5 배포 전 확인 항목

- 강수강도–골든타임 변환 규칙의 과학적 타당성 검토(기상 전문가 멘토링 활용)
- 가상 데이터의 실제 기상청 관측 API 교체
- 실제 침수 사례 각색의 정확성 확인

## 6. 체험 요소 및 학습 효과 (체험·참여형 설계, 학습 효과)

### 6.1 학습 목표 (관찰 가능한 동사로 진술)

1. 이동 거리·이동 수단·경로 위험도를 고려해 대피 소요 시간을 계산할 수 있다
2. 강수강도가 만든 골든타임과 자신의 대피 시간을 비교해 안전한 경로를 판단할 수 있다
3. "가장 가까운 길이 항상 안전하지 않다"는 원칙을 근거를 들어 설명할 수 있다
4. 호우 시 피해야 할 위험 요소(침수 도로, 지하공간, 하천변, 맨홀, 감전 위험 시설)를 식별할 수 있다

### 6.2 체험 흐름 (PBL 5단계)

| 단계 | 학습자 조작 | 즉각 반응(피드백) |
|---|---|---|
| STEP 1 상황 | 호우경보 상황판 확인 | "가장 가까운 길로 뛰면 될까?" 인지 충돌 질문 |
| STEP 2 준비 | 대피 전 준비 행동 선택(가스 밸브·전기 차단·비상용품 등) | 선택 즉시 골든타임 차감 — 준비할수록 안전하지만 시간 소모 |
| STEP 3 경로 선택 | 갈림길마다 3지선다 순차 선택(침수 지하차도 / 우회로 / 하천변 등) | 선택 즉시 안전·위험 판정과 잔여 시간 변화, 위험 경로는 즉시 실패 |
| STEP 4 결과·설명 구성 | 잔여 시간에 따른 도착 등급 확인 후 자기 언어로 정리 | "가장 가까운 길은 몇 분 걸릴 줄 알았지만 침수 때문에…" 빈칸 서술 |
| STEP 5 평가 | 총괄평가 3문항 풀이 | 문항별 즉시 채점·재시도 안내 |

### 6.3 양방향 상호작용 설계

모든 선택은 즉각적 결과(시간 증감, 안전·위험 판정)로 되돌아오며, 골든타임 카운터와 물 차오름 게이지가 상시 표시되어 시간 압박을 시각화한다. 준비 단계의 "안전 대 시간" 트레이드오프, 갈림길의 "빠른 길 대 안전한 길" 트레이드오프로 정답 암기가 아닌 판단 훈련이 이루어진다. 기상청 호우 행동요령의 실제 수칙이 게임 규칙과 피드백 문구로 변환되어, 플레이 자체가 안전 교육이 된다. 또한 읽기 속도가 느리거나 시간 압박에 취약한 학습자를 위해 타이머 일시정지, 텍스트 크기 조절 등 접근성 옵션을 제공한다.

### 6.4 평가 설계 (학습 목표별 이원화)

1. 시간 계산 문항: 수업에서 다루지 않은 새 상황(다른 강수강도, 다른 경로 구성)에서 대피 시간을 계산 — 정답이 명확한 수치 문항이므로 자동 채점의 신뢰성이 확보됨 (목표 1·2, 전이 측정)
2. 위험 식별 문항: 여러 장면 중 "가면 안 되는 곳"을 모두 선택 — 행동요령 기반 복수 선택, 자동 채점 (목표 4)
3. 원칙 서술 문항: "가장 가까운 길이 항상 안전하지 않은 이유"를 서술 — 정오 판정이 아닌 핵심 요소 체크리스트(거리·시간·침수 언급 여부)로 확인하고, 논리의 타당성은 교사 확인을 안내 (목표 3)

학습 목표의 성격에 따라 자동 채점과 체크리스트 채점을 구분하여, 측정한다고 주장하는 것과 실제로 측정되는 것의 간극을 없앤다.

## 7. 창의성 및 차별성 (아이디어 독창성)

### 7.1 재미와 개념의 일치

기존 게임형 교육 콘텐츠의 흔한 한계는 게임 보상(포인트·배지)이 학습 개념과 분리되어 장식으로 남는 것이다. 본 개발물은 모든 재미 요소가 학습 개념 그 자체가 되도록 설계했다. 물 차오름 애니메이션은 시간 압박의 시각화이고, 카운트다운은 누적 시간 계산이며, 위험 경로 경고는 행동요령 학습이고, 도착 등급은 시간 여유도 판단이다. 개념과 무관한 XP·배지 시스템은 의도적으로 배제했다.

### 7.2 실데이터가 게임을 움직이는 숨은 엔진

기상 데이터를 화면에 전시하는 대신, 강수강도 관측값이 골든타임을 결정하는 게임 엔진으로 작동시킨다. 학습자는 데이터를 "읽는" 것이 아니라 데이터가 만든 조건 안에서 "판단"하며, 이 과정에서 기상 관측값이 재난 위험으로 번역되는 인과를 자연스럽게 체득한다.

### 7.3 검증된 해외 사례의 선별 수용

일본 「쓰나미 대피 마스터」에서 복잡한 공식 없이 시간 계산을 직관화하는 "이동 = 시간" 은유를, UNDRR 「Stop Disasters!」에서 제한 조건 하 트레이드오프 판단 구조만을 선별 채택했다. Stop Disasters!의 복잡한 예산·건설 시스템과 채점 체계는 45분 수업이라는 현장 조건에 맞지 않아 배제하고, 점수 체계를 "대피 소요 시간"과 "안전 여부" 두 가지로 단순화했다.

### 7.4 정오가 하나로 수렴하지 않는 판단형 문제

정답 경로가 하나가 아니다. 골든타임 안에 안전하게 도착하는 경로가 복수이며, 준비를 얼마나 할지에 따라 같은 경로도 결과가 달라진다. 학습자는 정답을 찾는 것이 아니라 자신의 판단에 근거를 만드는 훈련을 하며, 이것이 실제 재난 상황에서 요구되는 역량과 일치한다.
