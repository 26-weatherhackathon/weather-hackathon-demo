# 우리 마을 홍수 방재 게임 — 개발계획서

기상·기후 AI 해커톤 경진대회 개발계획서 양식(양식2)에 따라 작성한 대표 데모 개발 계획. 각 장은 공고 심사 5개 항목(주제 적합성 20 / 체험·참여형 설계 25 / 학습 효과 10 / 아이디어 독창성 15 / 실현 가능성 30)에 대응한다.

## 1. 프로젝트명

우리 마을 홍수 방재 게임 — 국토지리정보원 수치표고모형(DEM)으로 만든 우리 마을 지형 위에서, 비가 내릴 때 물이 어디까지 차올라 어디가 잠기는지 직접 보고, 한정된 예산으로 방재 시설을 배치해 마을을 지키는 초등학생용 웹 기반 교육 게임

## 2. 개발 목적 및 필요성 (주제 적합성)

### 2.1 배경: 초등 재난 안전 교육의 한계

집중호우로 인한 도시·마을 침수는 한국에서 매년 인명 피해가 발생하는 대표적 기상 재난이다. 그러나 초등 대상 안전 교육은 영상 시청·수칙 암기 중심의 일방향 전달이어서, "왜 우리 동네 저지대가 먼저 잠기는가", "물은 어느 쪽으로 흐르는가" 같은 지형과 물의 관계를 눈으로 이해하는 경험을 주지 못한다. 초등 학습자에게는 등고선·강수강도 같은 추상 개념보다, 색으로 구분된 지형과 흐르는 물을 직접 보고 만지는 구체적 경험이 필요하다.

### 2.2 목적

실제 지형 데이터(DEM)가 만들어내는 물의 흐름과 침수를 눈으로 관찰하고, 학습자가 스스로 마을을 지키는 결정을 내리는 체험을 통해 다음을 달성한다.

- 땅의 높낮이가 물의 흐름과 침수를 결정한다는 인과를 색과 애니메이션으로 이해
- "낮은 곳이 먼저 잠긴다", "물은 높은 곳에서 낮은 곳으로 흐른다"는 원리의 체득
- 홍수 대비 행동(높이기·모래주머니·대피)의 효과를 직접 실험하며 학습

### 2.3 교육학적 기반

Piaget의 구체적 조작기(초등 4~6학년) 인지 발달 단계에 맞춰 설계한다. 이 시기 학습자는 구체적 사물·상황에 대해서는 논리적으로 사고할 수 있으나 추상적 기호(등고선, 수치 강수강도)는 아직 어려우므로, 아래 원칙을 따른다.

| 원칙 | 구현 |
|---|---|
| 대비 단계 시간 제한 없음 | 시설을 배치하는 대비 단계에는 제한 시간이 없어, 시간 압박·실패 불안 없이 충분히 탐구하고 결정한다. 호우 단계는 학습자가 직접 시작하며, 물이 차오르는 과정은 약 50초에 걸쳐 관찰 가능한 속도로 재생된다 |
| 추상 개념 없음 | 등고선 대신 DEM 색상 그래디언트로 높낮이를 직관화 |
| 큰 버튼 몇 개 | 복잡한 메뉴 없이 방재 시설 4종(모래주머니·제방·배수펌프·빗물 저류조)과 철거 도구를 큰 버튼으로 선택 |
| 즉시 피드백 | 시설 배치·저류조 반영이 즉시 수위 게이지·지형에 반영되어 인과를 즉각 확인 |
| 누적 점수 대신 안전 중심 결과 | 추상적 누적 점수 대신 "침수된 건물 수"와 별점(★)으로 마을 안전 결과를 표현한다. 예산은 경쟁적 점수가 아니라 "어디에 먼저 대비할지" 우선순위를 판단하게 하는 학습 제약 장치로 쓴다 |

## 3. 사용자 및 개발 형태

### 3.1 사용자

초등학교 4~6학년(단일 버전). 학년별 교육과정 연계는 6.5장 참조. 시간 압박이나 복잡한 수치 계산 없이, 지형을 보고 마을을 지키는 결정을 반복하며 배우도록 설계한다.

### 3.2 개발 형태

게임형 웹 인터랙티브 시뮬레이션. 설치·로그인 없이 URL 접속만으로 심사위원·교실 기기에서 즉시 시연 가능한 단일 웹앱. 한 차시(20~30분) 안에 여러 번 반복 플레이가 가능한 분량.

## 4. 활용 데이터 및 과학 개념

### 4.1 활용 데이터

두 개의 공공데이터를 결합한다.

**(1) 국토지리정보원 5m 수치표고모형(DEM) — 핵심 데이터**

- 출처: 국토지리정보원 국토정보플랫폼(geoinfo.go.kr) / 공공데이터포털(data.go.kr), 공공데이터 CC-BY, 무료
- 해상도: 5m × 5m — 학교·마을 단위 지형을 식별할 수 있는 정밀도
- 포맷: GeoTIFF를 JSON 격자로 변환하여 게임에 삽입
- 활용 파이프라인:
  - 색상 맵핑: 고도에 따라 색을 입힘(높은 곳 100m↑ 갈색 / 낮은 곳 10m↓ 파랑)
  - 지형 생성: 우상단 산악, 중앙 대각선 하천, 하천 주변 저지대 마을(집·학교)이 드러나도록 30×30 격자를 결정적 시드로 구성(같은 시드면 항상 같은 지형 → 재현 가능)
  - 침수 모델링("수위 상승"): 호우 시나리오의 예상 최고 수위까지 물이 시간에 따라 차오르고, (셀 고도 + 방재 시설로 높인 높이)가 현재 수위보다 0.3m 이상 낮으면 침수로 판정. 물이 낮은 곳부터 채워지는 인과를 시간에 따른 수면 상승으로 보여준다

**(2) 기상청 강수강도(RN_INT, mm/h) — 강우 조건**

- 출처: 기상자료개방포털(data.kma.go.kr) API
- 역할: 얼마나 많은 비가 내리는지(강우량)를 결정하여 침수 기준선을 올리는 입력값. 화면에는 "비가 많이 와요"처럼 초등 눈높이 표현으로 노출하고, 내부적으로는 API 응답과 동일한 필드 구조를 유지해 실데이터 연동을 보장한다.

### 4.2 과학 개념

땅의 높낮이(지형)와 물의 흐름, 저지대의 침수 위험, 강우량과 침수의 관계, 물의 순환(비 → 지표 흐름 → 저지대 고임).

## 5. 적용 기술 및 실현 구체성 (실현 가능성)

### 5.1 구현 방식과 기술 스택

- 프론트엔드: Next.js(App Router) + TypeScript + Tailwind 기반 단일 웹앱. 지형도는 아이소메트릭(isometric) 뷰로 렌더링한다. 자유 회전 3D 대비 구현 복잡도가 낮으면서도, 고도차가 만드는 입체감으로 "낮은 곳이 먼저 잠긴다"는 인과를 2D 탑뷰보다 직관적으로 전달한다. DEM 색상화 지형도, 시간에 따라 차오르는 물(수면) 렌더, 침수 애니메이션(비·번개·물보라 이펙트), 방재 시설 팔레트 UI를 표준 웹 기술로 구현
- 데이터 계층: 국토지리정보원 DEM(GeoTIFF→JSON 격자)과 기상청 강수강도 API 응답 스키마와 동일한 구조의 데이터 객체를 두고, ① 고도→색상 맵핑(`classifyAltitude`/`getColorByAltitude`), ② 호우 시나리오 수위 곡선(예상 최고 수위까지 시간에 따라 상승), ③ 침수 판정((고도 + 시설 높이) < 수위 − 0.3m)을 각각 단일 함수로 분리. 가상 데이터에서 실제 API·DEM 파일로 교체가 한 지점에서 이루어지는 구조
- 배포: 정적/서버리스 호스팅 기반 상시 접속 URL 제공

### 5.2 UI 레이아웃

렌더링 방식은 아이소메트릭(isometric) 뷰로 명시한다. 2D 탑뷰는 고도차를 색으로만 표현해 물 흐름 방향의 인과가 약하고, 자유 회전 3D는 카메라 조작이 추가되어 초등 학습자에게 인지 부담을 주고 구현 공수도 크다. 아이소메트릭은 고정된 시점에서 고도차가 입체적으로 드러나 "높은 곳에서 낮은 곳으로 물이 흐른다"를 별도 설명 없이 보여준다.

화면 박스 배치·버튼 위치·결과 표시 위치 등 구체적 레이아웃은 구현 단계에서 결정한다. 각 선택 버튼의 기능적 효과(DEM 반영 규칙)는 5.3장에, 화면에 쓰이는 색상 축은 아래 5.2.1에 규정한다.

#### 5.2.1 색상 체계

지형 고도와 선택 결과(안전·경고·위험)를 서로 다른 색상 축으로 분리해, 두 정보가 화면에서 겹쳐도 혼동되지 않도록 한다. 지형은 저채도 자연색(갈색·초록·파랑 계열), 결과 피드백은 고채도 신호색(초록·주황·빨강)을 사용해 축을 명확히 구분한다.

**(1) 지형 고도 색상 (DEM 그래디언트)**

| 고도 구간 | 색상 | 의미 |
|---|---|---|
| 100m 이상 | 갈색 (#8D6E4A) | 산·고지대 |
| 30~100m | 연두 (#A9C97E) | 완만한 경사지 |
| 10~30m | 초록 (#6FA96F) | 평지·마을 |
| 10m 미만 | 파랑 (#4A90D9) | 저지대·계곡·물 |

**(2) 선택 결과 안전 신호 색상**

| 상태 | 색상 | 적용 대상 |
|---|---|---|
| 안전 | 초록 (#43A047) | 침수선 위로 올라간 지역, "모두 안전해요" 결과 |
| 경고(부분 대비) | 주황 (#FB8C00) | 모래주머니 등으로 일부만 막은 지역, 강한 비에는 넘칠 수 있음 |
| 위험(침수) | 빨강 (#E53935) | 침수 판정된 지역, "○○지역이 잠겼어요" 결과 |

지형 색상(자연색 계열)과 결과 신호색(신호등 계열)을 색상 축으로 분리했기 때문에, 파란 저지대가 안전 신호의 초록과 섞이지 않는다. 색약 학습자를 고려해 신호색에는 색상 외에 아이콘(안전 ✓ / 경고 △ / 위험 ✕ 형태의 기호, 배포 시 이모지 대신 SVG 아이콘 사용)을 함께 표시한다.

### 5.3 방재 시설별 DEM·수위 반영 규칙

플레이어는 대비 단계에서 시작 예산(800원) 안에서 방재 시설을 지형 타일에 배치한다(하천 등 저지대 물 타일에는 설치 불가). 호우가 시작되면 물이 예상 최고 수위까지 차오르고, 각 타일의 (고도 + 시설로 높인 높이)가 그 시점 수위보다 0.3m 이상 낮으면 침수로 판정된다. 배치·철거는 즉시 수위 게이지·지형에 반영된다.

| 시설 | 비용 | DEM·수위 효과 | 학습 포인트 |
|---|---|---|---|
| 모래주머니 | 40원 | 타일 고도 +1.5m | 조금 낮은 집을 값싸게 올려 막는다 |
| 제방 | 120원 | 타일 고도 +6m | 많이 낮은 집도 든든하게 물 위로 올린다 |
| 배수펌프 | 150원 | 타일 강제 배수(고도와 무관하게 침수 제외) | 높이로 못 막는 낮은 곳의 물을 퍼낸다 |
| 빗물 저류조 | 130원 | 마을 전체(지역) 수위 −0.5m(누적 상한 −2.5m) | 빗물을 저장해 마을 전체 물 높이를 낮춘다 |
| 철거 | 0원(설치비 50% 환급) | 배치한 시설 제거 | 예산을 회수해 배치를 바꾼다 |

이 규칙으로 "낮은 곳이 잠긴다", "조금/많이 낮은 집에는 알맞은 방법이 다르다", "높이로 못 막으면 물을 뺀다", "지역 전체 대응(저류조)"이라는 학습 목표가 모두 게임 규칙 안에서 작동한다. 예산 제약은 "한정된 자원으로 어디를 먼저 지킬지"를 판단하게 하여 실제 재난 대비의 우선순위 결정 역량과 연결된다.

### 5.4 AI 도구 활용 방식

- 설계 단계: 생성형 AI로 학습 목표–결과 표현 정합성 검토, 초등 눈높이 문구(피드백·질문) 초안 생성
- 개발 단계: AI 페어 코딩으로 DEM 처리(D8·침수 판정)·게임 로직·UI 구현
- 검증 단계: DEM 격자에서 선택별 침수 결과가 의도한 학습 목표대로 나오는지 사전 시뮬레이션

## 6. 체험 요소 및 학습 효과 (체험·참여형 설계, 학습 효과)

### 6.1 학습 목표 (관찰 가능한 동사로 진술)

1. DEM 색상 지형도를 보고 우리 마을에서 물이 어디로 흐르고 어디가 먼저 잠기는지 말할 수 있다
2. 땅의 높낮이가 침수를 결정한다는 원리를 근거를 들어 설명할 수 있다
3. 마을을 지키기 위한 방재 시설(모래주머니·제방·배수펌프·빗물 저류조)을 예산 안에서 선택·배치하고 그 결과를 예측할 수 있다
4. "낮은 곳이 먼저 잠긴다"는 원칙을 우리 동네 상황에 적용해 설명할 수 있다

### 6.2 체험 흐름 (3단계)

| 단계 | 내용 |
|---|---|
| 1 대비(배치) | 지형도를 보며 예산(800원) 안에서 방재 시설을 위험한 집 타일에 배치. 제한 시간이 없어 수위 게이지로 "물 높이 vs 마을 고도"를 비교하며 어디가 위험한지 충분히 탐색·조정한다 |
| 2 호우 | "⛈️ 폭우 시작"을 누르면 물이 예상 최고 수위까지 차오른다(약 50초). 어떤 집이 잠기고 어떤 방재가 막아내는지 실시간으로 관찰 |
| 3 결과 | 침수된 건물 수 + 별점(★ 0~3) + 남은 예산 → "다시 도전하기"로 배치를 바꿔 반복 실험 |

대비 단계는 정해진 순서로 흐르는 애니메이션이 아니라 학습자 주도 탐색이다. 시설을 배치·철거하고 수위 게이지를 보며, 같은 상황을 몇 번이든 다시 꾸며 비교할 수 있다. 제한 시간이 없으므로 학습자는 수위와 고도를 스스로 비교해 충분히 관찰하고, 시간 압박이나 실패 불안 없이 "높낮이 → 침수 위험 → 알맞은 방재"의 인과를 능동적으로 탐구한다. 호우 단계의 수위 상승은 학습자가 직접 시작하고 관찰 가능한 속도로 재생되어, 자신의 대비가 어떻게 작동하는지 눈으로 확인한다.

### 6.3 양방향 상호작용 설계

상호작용은 두 층위로 설계한다. 첫째, 대비 단계에서 학습자는 수위 게이지로 "현재/예상 수위 vs 마을 고도"를 비교하며 위험한 집을 스스로 찾고, 방재 시설을 배치·철거해 결과가 어떻게 달라지는지 반복 실험한다. 제한 시간이 없어 같은 배치를 몇 번이든 바꿔 확인할 수 있어 2.3장의 구체적 조작기 원칙(직접 조작을 통한 학습·충분한 관찰)에 부합한다. 둘째, 배치·저류조 반영은 즉시 수위 게이지·지형에 되돌아오고, 호우 결과는 "침수된 건물 ○채"와 별점(★)으로 표현된다. 누적 점수·배지 대신 마을 안전(침수 건물 수)으로 결과를 나타내 초등 학습자가 직관적으로 이해한다. 학습자는 "모래주머니"와 "제방", "배수펌프"와 "빗물 저류조"의 차이를 직접 실험하며, 플레이 자체가 지형–물–침수 관계에 대한 안전 교육이 된다. 접근성 측면에서 대비 단계의 시간 제한 부재로 시간 압박이 제거되며, 큰 버튼·색상 대비 등을 제공해 읽기 속도가 느리거나 불안에 취약한 학습자도 함께 참여할 수 있도록 한다.

### 6.4 평가 설계 (학습 목표별)

1. 지형 읽기: 새 지형도를 보여주고 "어디가 먼저 잠길까?"를 물어 저지대 식별 능력 확인(목표 1·4, 자동 채점)
2. 대비 행동 예측: 특정 선택 후 결과를 예측하게 하여 인과 이해 확인(목표 3, 자동 채점)
3. 원리 설명: "낮은 곳이 왜 먼저 잠기는지"를 자기 언어로 말/서술하게 하고, 핵심 요소(높낮이·물 흐름 언급 여부) 체크리스트로 확인하며 교사 확인을 안내(목표 2)

자동 채점이 신뢰성 있게 가능한 문항과 교사 확인이 필요한 서술을 구분하여, 측정한다고 주장하는 것과 실제로 측정되는 것의 간극을 없앤다.

- 게임 10~15분: 개별 플레이 3~4회 반복 → 반 전체 투표로 최선의 선택 논의

- 정리 2~3분: "홍수는 미리 준비하면 막을 수 있어요"로 마무리

## 7. 창의성 및 차별성 (아이디어 독창성)

### 7.1 재미와 개념의 일치

기존 게임형 교육 콘텐츠의 흔한 한계는 게임 보상(포인트·배지)이 학습 개념과 분리되어 장식으로 남는 것이다. 본 개발물은 모든 재미 요소가 학습 개념 그 자체가 되도록 설계했다. 색으로 물든 지형은 높낮이의 시각화이고, 차오르는 물은 낮은 곳부터 잠기는 원리이며, 침수 애니메이션은 저지대 위험의 학습이고, 예산 안에서 마을을 지키는 방재 배치는 대비 행동과 우선순위 판단의 실험이다. 개념과 무관한 누적 점수·배지 시스템은 두지 않았고, 결과는 마을 안전(침수된 건물 수)과 그에 대응하는 별점으로만 표현한다. 예산 역시 경쟁 점수가 아니라 "한정된 자원으로 어디를 먼저 지킬지"를 판단하게 하는 학습 장치로, 재미와 개념이 분리되지 않도록 설계했다.

### 7.2 실데이터가 게임을 움직이는 숨은 엔진

지형 데이터를 화면에 전시하는 대신, 국토지리정보원 5m DEM 실측 고도가 수위 상승에 따른 침수를 결정하는 게임 엔진으로 작동한다. 학습자는 지도를 "읽는" 것이 아니라 실제 우리 마을 지형이 만든 조건 안에서 마을을 지키며, 이 과정에서 지형이 재난 위험으로 번역되는 인과를 자연스럽게 체득한다.

### 7.3 초등 인지 발달에 맞춘 과감한 단순화

중학생용 시간 압박·수치 계산 게임의 관성을 따르지 않고, Piaget 구체적 조작기에 맞춰 대비 단계의 제한 시간과 누적 점수, 추상 기호를 걷어냈다. 등고선 대신 색 그래디언트, 강수강도 수치 대신 초등 눈높이 표현, 복잡한 메뉴 대신 방재 시설 몇 개로 단순화하여, 초등 학습자가 인지 부담 없이 지형–물–침수의 핵심 원리에 집중하도록 했다.

### 7.4 정답이 하나로 수렴하지 않는 판단형 놀이

정답 선택이 하나가 아니다. 지형과 강우량에 따라 마을을 지키는 방법이 여럿이며, 어느 지역을 먼저 보호하느냐에 따라 결과가 달라진다. 학습자는 정답을 맞히는 것이 아니라 여러 번 시도하며 "왜 이렇게 될까?"를 스스로 탐구하고, 이것이 실제 재난 대비에서 요구되는 판단 역량과 일치한다.
