# 「예보관 의사결정 체험」 계획 개정 v1.1 — 가이드 우선 (PhET보다 쉽게)

> Build Spec v1.0의 **델타(변경분)** 문서. v1.0과 충돌하는 부분은 이 문서가 우선한다.
> 대상: 기상·기후 AI 해커톤 2026 (한국생산성본부). 타깃 = 중3~고1 + 일반 시민, 전공지식 0.

---

## 0. 한 줄 요약 (무엇이 바뀌나)

PhET식 **자유 탐색 위에 통계 개념(Brier·확률)을 얹은 체험**에서 →
**앱이 직접 손을 잡아주는 자기완결적 가이드 학습**으로.

- 확률적 사고는 **살린다.** 단, 화면에는 숫자(Brier) 대신 **'확신의 정도' + 쉬운 말 피드백.**
- 별도 교사/진행자 자료 **0개.** 앱의 온보딩·튜토리얼·인라인 설명이 그 역할을 전부 흡수한다.
- "PhET보다 쉽게" = 빈 캔버스에 던져두지 않고, **다음에 뭘 할지 항상 보이게** 한다.

## 1. 설계 원칙 (5)

| # | 원칙 | 구현 신호 |
|---|---|---|
| P1 | **손을 잡아준다** | 모든 단계에 "지금 할 일" 한 줄 배너 + 봐야 할 곳 스포트라이트/화살표 |
| P2 | **점수 대신 피드백** | Brier는 내부 계산만. 화면엔 '확신 게이지' + 서사형 피드백("거의 맞혔어요") |
| P3 | **먼저 보여주고 시킨다** | 점수 없는 **튜토리얼 케이스**로 조작·용어를 1회 완전 시연 |
| P4 | **용어는 그 자리에서** | 위성·레이더·모델·관측마다 인라인 한 줄 설명 상시 노출 |
| P5 | **자기완결** | 진행자 없이 첫 방문자가 끝까지 완주. 별도 교육자료 불필요 |

## 4. 채점 → 피드백 변환 (확률 개념을 '숫자 없이' 체감)

**내부 엔진은 그대로 유지** (Build Spec §6 / app-plan §5: Brier + 형태 + 기온).
그 위에 **표시 레이어**를 새로 둔다.

### 4.1 입력: 확신 게이지 (확률 슬라이더의 옷 갈아입히기)
- 0~100 값은 동일하되 라벨을 말로: `반반 같아요(40~60) → 비가 올 듯(60~80) → 거의 확실해요(80~100)`.
- 슬라이더를 끌면 게이지 색·표정이 함께 변해 '확신의 무게'를 촉각적으로 전달(Spec §5-C C4 재활용).

### 4.2 출력: 서사형 피드백 (점수 대신)
점수 구간 → 한 줄 서사로 매핑(`lib/feedback.ts` 신설):

| 총점 | 표시 | 예시 문구 |
|---|---|---|
| 85~100 | ✅ 잘 봤어요 | "레이더 에코가 결정적 단서였죠. 정확했어요." |
| 60~84 | 🟢 좋아요 | "방향은 맞았어요. 조금만 더 확신을 가져도 됐겠네요." |
| 35~59 | 🟡 아쉬워요 | "비구름을 한 발 늦게 봤어요. 레이더를 먼저 볼까요?" |
| 0~34 | 🔁 다시 보기 | "이번엔 빗나갔어요. 어디서 갈렸는지 같이 볼게요." |

- **'왜 이 점수일까?'** 를 펼치면 그때 Brier·구간·반사실("확신을 70%로 했다면 더 좋았어요")을 보여준다 → 확률 개념을 *원하는 사람만* 깊게.

## 5. 가이드/튜토리얼 시스템 (앱이 교육자료를 대체)

| 요소 | 동작 |
|---|---|
| **다음 할 일 배너** | 화면 상단 고정 1줄: "레이더를 보고 비구름 위치를 찾아보세요" |
| **코치마크** | 스포트라이트 + 말풍선으로 클릭할 곳을 순서대로 안내(튜토리얼·첫 케이스) |
| **인라인 용어칩** | 위성/레이더/모델/관측 호버·탭 시 "지금 비가 내리는 위치" 같은 1줄 + 미니 예시 |
| **막힘 방지 힌트** | 일정 시간 무동작 시 다음 행동을 살짝 노출 |
| **접근성** | 모든 가이드는 키보드/스크린리더로도 동일 작동, `prefers-reduced-motion` 대응 |

> 튜토리얼 케이스는 "따라 하면 자동으로 다음 단계"로 진행돼, 처음 본 사람도 조작을 몸으로 익힌다.
